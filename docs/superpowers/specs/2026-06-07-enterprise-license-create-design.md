# 企业证照新增功能重设计

## 背景

当前企业证照管理已有新增弹窗、图片上传接口和营业执照/食品经营许可证 OCR 接口，但实际体验较差：上传、识别、核对和手工录入混在同一弹窗中，证照类型需要用户先判断，OCR 原始结果暴露过重，字段也没有严格围绕两类核心证照设计。

本设计将“新增证照”重做为企业端的上传优先流程，仅覆盖两类企业证照：

- 营业执照
- 食品经营许可证

OCR 是辅助录入能力。最终保存数据必须以用户确认后的表单值为准。

## 目标

- 让用户优先上传证照原件，再由系统自动识别并填充字段。
- 自动判断证照类型，但允许用户改正。
- OCR 失败时不阻断新增，用户可以基于已上传原件手工补齐字段。
- 同一企业同一证照类型使用版本化管理，只保留一张当前版本，旧版本转为历史。
- 当前版本参与红黄牌预警，历史版本不参与预警。

## 非目标

- 不覆盖学校和食堂证照新增。
- 不支持除营业执照、食品经营许可证以外的企业证照类型。
- 不引入证照审批流。
- 不要求 OCR 达到自动保存精度。
- 不保存或展示大段 OCR 原始 JSON 作为普通用户界面。

## 推荐方案

采用三步式新增向导，可以实现为宽弹窗或抽屉式向导。

### 第一步：上传原件

用户点击“新增证照”后进入上传页。该页只承担上传任务：

- 支持 JPG、PNG、PDF。
- 单文件不超过 10MB。
- 上传成功后显示证照预览。
- 系统自动识别证照类型并给出建议：营业执照或食品经营许可证。
- 用户可以手动切换证照类型。

如果上传失败，停留在当前步骤并展示失败原因。

### 第二步：识别核对

系统根据证照类型抽取字段，进入核对表单。

推荐布局：

- 左侧固定显示证照原件预览。
- 右侧显示字段表单。
- OCR 填入的字段显示“已识别”。
- 空字段显示“需补充”。
- 用户修改过的字段显示“已修改”。
- 提供“查看识别文本”折叠区，仅展示识别文本，不展示完整 JSON。

OCR 失败时，保留已上传文件，直接进入手工录入表单，并提示用户补齐必填字段。

用户切换证照类型时，系统按新类型重新解析字段。已经手工修改过的字段不得静默覆盖，应提示用户确认。

### 第三步：确认保存

保存前展示摘要：

- 证照类型
- 证照编号
- 企业名称
- 有效期止或长期有效
- 是否会替换当前同类型证照

如果同一企业已有同类型当前证照，保存前提示：新证照保存后将成为当前版本，原证照转为历史版本。

## 字段设计

### 共用字段

| 字段 | 要求 | 说明 |
| --- | --- | --- |
| 所属企业 | 必填，系统带出 | 企业用户默认当前企业，不允许选择其他企业 |
| 证照类型 | 必填 | 仅允许营业执照、食品经营许可证 |
| 证照编号 | 必填 | 营业执照为统一社会信用代码；食品经营许可证为许可证编号 |
| 企业名称 | 必填 | 与企业档案名称不一致时允许保存，但必须二次确认 |
| 法定代表人/负责人 | 必填 | 营业执照显示法定代表人；食品经营许可证显示法定代表人/负责人 |
| 住所/经营场所 | 必填 | 按证照类型调整字段文案 |
| 有效期起 | 可空 | 识别到时填入 |
| 有效期止 | 条件必填 | 食品经营许可证必填；营业执照可选择长期有效 |
| 长期有效 | 条件字段 | 仅营业执照可用 |
| 发证机关 | 选填 | OCR 识别到时填入 |
| 发证日期 | 选填 | OCR 识别到时填入 |
| 证照文件 URL | 必填 | 上传成功后写入 |

### 营业执照特有字段

| 字段 | 要求 |
| --- | --- |
| 经营范围 | 强提示补全 |
| 成立日期 | 选填 |
| 注册资本 | 选填 |

### 食品经营许可证特有字段

| 字段 | 要求 |
| --- | --- |
| 主体业态 | 必填 |
| 经营项目 | 必填 |
| 日常监督管理机构 | 选填 |

## 数据状态

企业证照需要增加或规范以下状态字段：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| status | current/history/voided | 当前版本、历史版本、已作废 |
| version | 1 | 同企业同类型递增 |
| replacedBy | el_xxx | 历史版本被哪张新证照替换 |
| replacedAt | ISO 时间 | 转为历史版本时间 |
| voidedAt | ISO 时间 | 作废时间 |
| voidReason | 文本 | 作废原因 |
| isLongTerm | true/false | 营业执照是否长期有效 |
| ocrSource | business-license/food-license/manual | 数据来源提示，不作为最终依据 |

现有列表应默认展示 `status=current` 的证照，并提供查看历史版本入口。

## OCR 规则

### 类型判断

自动判断结果作为建议值，不直接锁定。判断参考：

- 包含“营业执照”“统一社会信用代码”等关键词，倾向营业执照。
- 包含“食品经营许可证”“许可证编号”“主体业态”“经营项目”等关键词，倾向食品经营许可证。

如果判断不明确，默认停留在“请选择/请确认类型”状态，用户确认后再解析字段。

### 字段抽取

营业执照重点抽取：

- 统一社会信用代码
- 企业名称
- 法定代表人
- 住所
- 经营范围
- 成立日期
- 营业期限
- 登记机关或发证机关
- 发证日期

食品经营许可证重点抽取：

- 许可证编号
- 经营者名称
- 法定代表人或负责人
- 住所或经营场所
- 主体业态
- 经营项目
- 有效期
- 发证机关
- 发证日期

## 校验规则

- 证照文件必填。
- 证照类型必填，仅允许营业执照、食品经营许可证。
- 证照编号必填。
- 企业名称必填。
- 企业名称与企业档案名称不一致时，必须二次确认。
- 食品经营许可证有效期止必填。
- 营业执照支持长期有效；如果填写具体日期，有效期止不得早于有效期起。
- 食品经营许可证有效期止不得早于有效期起。
- 同一企业、同一证照类型、同一证照编号重复上传时提示“可能重复”，由用户确认是否作为新版本保存。

## 版本化规则

- 同一企业、同一证照类型，只允许一张当前版本。
- 新增同类型证照时，新证照保存为 `current`。
- 原当前证照转为 `history`，记录 `replacedBy` 和 `replacedAt`。
- 历史版本不得进入红黄牌预警统计。
- 删除操作不物理删除，改为作废或停用。
- 当前版本作废后，该企业该类型证照显示为缺失或无当前有效版本。

## 预警规则

只对当前版本计算预警：

- 已过期或有效期小于等于 7 天：红牌。
- 有效期小于等于 30 天：黄牌。
- 有效期大于 30 天：绿牌。
- 营业执照长期有效：显示长期有效，不进入到期预警。
- 历史和作废证照不参与预警统计。

## 接口影响

现有接口可演进，不要求一次性拆成全新服务。

### 上传接口

`POST /api/enterprise-licenses/upload`

调整点：

- 文件大小限制从当前 5MB 调整为 10MB。
- 允许 JPG、PNG、PDF；不再建议 GIF。
- 返回文件 URL、文件名、MIME 类型、文件大小。

### OCR 接口

建议新增统一识别入口：

`POST /api/enterprise-licenses/ocr`

请求：

```json
{
  "imageUrl": "/uploads/license-images/license_xxx.png",
  "preferredType": "营业执照"
}
```

返回：

```json
{
  "suggestedType": "营业执照",
  "confidence": "medium",
  "fields": {
    "licenseNo": "9135...",
    "enterpriseName": "某某公司"
  },
  "recognizedText": "..."
}
```

保留现有 `/api/ocr/business-license` 和 `/api/ocr/food-license` 作为内部实现或兼容入口。

### 保存接口

`POST /api/enterprise-licenses`

调整点：

- 后端根据登录用户确定企业 ID 和企业名称。
- 后端校验证照类型、必填字段、有效期规则。
- 后端执行版本化替换，不能依赖前端处理旧版本状态。
- 返回保存后的当前证照，以及被替换的旧证照 ID。

## 前端组件建议

- `EnterpriseLicenseCreateWizard`：三步式新增容器。
- `LicenseUploadStep`：上传和预览。
- `LicenseReviewStep`：原件预览 + 字段核对表单。
- `LicenseConfirmStep`：保存摘要和版本替换提示。
- `LicenseFieldStatus`：已识别、需补充、已修改状态标识。
- `LicenseHistoryPanel`：后续在详情或列表中查看历史版本。

## 错误处理

- 上传失败：停留上传步骤，提示失败原因。
- 文件格式或大小不合规：阻止上传并提示允许格式和大小。
- OCR 服务不可用：进入手工录入流程，提示“识别服务暂不可用，可手工完善后保存”。
- OCR 结果为空：进入手工录入流程。
- 保存校验失败：定位到对应字段，并保留用户已输入内容。
- 保存时发现同类型当前证照：进入确认替换提示。

## 验收标准

- 企业用户点击新增证照后，首先看到上传原件步骤。
- 上传营业执照或食品经营许可证后，系统能给出证照类型建议。
- 用户可以改正系统建议的证照类型。
- OCR 成功时，对应字段自动填入并标识为已识别。
- OCR 失败时，用户仍可基于已上传文件手工补齐并保存。
- 营业执照和食品经营许可证只展示各自需要的字段。
- 同一企业新增同类型证照时，旧证照自动转为历史版本，新证照为当前版本。
- 列表默认展示当前版本。
- 红黄牌预警只统计当前版本。
- 历史版本和作废版本不参与预警。

## 测试建议

- 上传 JPG、PNG、PDF 成功。
- 上传超 10MB 文件失败。
- 上传非允许格式失败。
- 营业执照 OCR 成功后进入核对页。
- 食品经营许可证 OCR 成功后进入核对页。
- OCR 失败后可继续手工保存。
- 切换证照类型时，不静默覆盖已修改字段。
- 食品经营许可证缺少有效期止不能保存。
- 营业执照长期有效可以保存且不触发到期预警。
- 同企业同类型新增第二张证照后，旧证照状态为历史，新证照状态为当前。
- 企业名称与档案名称不一致时触发二次确认。
